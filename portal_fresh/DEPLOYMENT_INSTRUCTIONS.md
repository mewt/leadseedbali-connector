# Google Sheet Setup Instructions

## 1. Set Up Your Google Sheet

Open your Google Sheet and add the following **column headers** in Row 1:

| A | B | C | D | E | F | G |
|---|---|---|---|---|---|---|
| **Date** | **Login Time** | **Guest Name** | **Email Address** | **Consent** | **Device ID (MAC)** | **Device Type** |

> **Note:** The sheet tab can be named anything. The script will try to find a tab called **"Leads"** first, and fall back to the **active sheet** if not found.

---

## 2. Update Google Apps Script

1. In your Google Sheet, click **Extensions** → **Apps Script**
2. **Delete all existing code** in the editor
3. **Copy and paste** the code from `google-apps-script.js`
4. Click **Save** (💾 icon)
5. Click **Deploy** → **New deployment**
6. Click the gear icon ⚙️ next to "Select type" → choose **Web app**
7. Set the following:
   - **Description**: "WiFi Portal Data Capture"
   - **Execute as**: **Me** (your email)
   - **Who has access**: **Anyone**
8. Click **Deploy**
9. **Copy the Web APP URL** (it will look like: `https://script.google.com/macros/s/.../exec`)

> **Note**: If you're using the same URL as before, you may need to **update the existing deployment** instead:
> - Click **Deploy** → **Manage deployments**
> - Click the edit (✏️) icon
> - Change **Version** to **New version**
> - Click **Deploy**

---

## 3. Update the Portal Code

Replace the old portal files with the updated ones in the `portal_fresh/` folder:
- `index.html`
- `js/index.js`
- `css/index.css`

The webhook URL is already configured inside `js/index.js`. If you created a **new** Apps Script deployment, replace the `WEBHOOK_URL` in `js/index.js` with your new URL.

---

## 4. Data Format

When a guest logs in, the following will be captured:

| Field | Source | Example |
|-------|--------|---------|
| **Date** | Auto-generated by script (GMT+8) | `2025-06-12` |
| **Login Time** | Auto-generated by script (GMT+8) | `14:30:25` |
| **Guest Name** | User input (validated) | `John Doe` |
| **Email Address** | User input (validated) | `john@example.com` |
| **Consent** | Checkbox (checked by default) | `TRUE` or `FALSE` |
| **Device ID (MAC)** | URL parameter from Ruijie router (`mac` or `userMac`) | `AA:BB:CC:DD:EE:FF` |
| **Device Type** | Auto-detected from browser user agent | `iOS`, `Android Phone`, `Android Tablet`, `Windows`, `Mac`, `Linux`, `Unknown` |

---

## 5. Form Validation

The portal enforces client-side validation before submission:

- **Name** must be at least 2 words (first + last name)
- Each name word must be at least 2 characters
- Names must start with a letter
- No numbers allowed in names
- Common junk names are blocked (e.g., `test test`, `asdf asdf`, `abc xyz`, `fake dummy`, `guest user`)
- **Email** must be a valid format

---

## 6. Testing

After deployment:
1. **Clear your browser cache** or test in an incognito window
2. Access the captive portal from a device on the guest WiFi
3. Enter name and email
4. Click "Connect Now"
5. Check your Google Sheet — a new row should appear with all 7 columns filled

## Troubleshooting

### Data not appearing in sheet?
- Check that the Apps Script is deployed with **"Anyone"** access
- Make sure you deployed a **New version** after updating the code
- Check that the sheet has the correct column order (Date, Login Time, Guest Name, Email, Consent, MAC, Device Type)
- **Clear browser cache** — old `index.js` may be cached by the browser or Ruijie Cloud CDN

### MAC address showing as empty?
- The MAC address must be passed by the Ruijie router in the portal redirect URL
- Check the URL parameters when accessing the portal (look for `mac` or `userMac`)
- If not available, this column will be blank

### Device Type showing as "Unknown"?
- The device type is detected from the browser's user agent string
- If it shows "Unknown", the browser/device is not in the recognition list

### Preview shows "Oops! Server Not Available"?
- The Ruijie Cloud dashboard preview does **not** have a real authentication context
- The portal only works fully when accessed from a device physically connected to the guest WiFi
- `/api/auth/general` returning 404 in preview is expected behavior

## Rebuild Zip Command

If you make local changes and need to regenerate `portal_fresh.zip`:

```bash
python3 -c "
import zipfile, os
zip_path = '/home/himehime/Archive/portal_fresh.zip'
src_dir = '/home/himehime/Archive/portal_fresh'
with zipfile.ZipFile(zip_path, 'w', zipfile.ZIP_DEFLATED) as zf:
    for root, dirs, files in os.walk(src_dir):
        for f in files:
            if f in ('.DS_Store', 'portal_fresh.zip', 'google-apps-script.js'):
                continue
            full = os.path.join(root, f)
            arcname = os.path.relpath(full, src_dir)
            zf.write(full, arcname)
print('Done:', zip_path)
"
```
